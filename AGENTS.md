# EmbodiedLab 项目协作规则

## 用户与协作方式

- 用户是西安交通大学自动化专业硕士生，目标岗位为具身智能算法实习生，关注 VLA、WAM 和动态物体操作。
- 用户正在学习独立项目开发与 Git/GitHub。执行重要操作时，用中文说明目的、结果和下一步，不假设用户熟悉命令行。
- 从求职作品、科研可复现性和个人电脑可落地性的角度协助用户。

## 项目目标

- 项目暂定名为 EmbodiedLab。
- 建设一个面向 VLA/WAM 实验的本地诊断与学习闭环平台。
- 核心能力包括实验配置比较、日志与指标可视化、episode 轨迹检查、action/world token 对齐诊断、轻量状态估计基线，以及学习计划和周报。
- 优先完成能公开演示、能写进简历、CPU 可运行的功能，不依赖本地训练大模型。

## 文件边界

- 项目根目录固定为本 Git 仓库根目录。所有新建项目文件、下载物、缓存、日志、测试产物和生成结果都必须位于根目录或其子目录中。
- 不得在项目根目录之外创建、覆盖、移动或删除文件，除非用户针对具体路径明确授权。
- 可以只读访问用户明确提供的外部文件，但处理结果必须写回项目根目录内。
- 临时文件使用 `tmp/`；项目数据使用 `data/`；生成报告使用 `outputs/`；不要把临时产物放到 C 盘。
- 必须写入 Codex、Git 或其他工具的用户级配置时，先说明原因和具体位置。

## 数据与隐私

- 不得提交简历中的电话、邮箱、证件、账号、密钥或其他个人敏感信息。
- 不得提交实习公司的私有代码、模型权重、数据、服务器地址、日志或未公开实验结果。
- 示例和测试优先使用合成数据或明确可公开的数据。
- 大模型权重、原始视频、checkpoint 和大规模 episode 数据不得进入 Git；应通过 `.gitignore` 排除。

## 工程约定

- 默认使用 Python 3.11+，优先选择轻量、跨平台、可在 Windows 本地运行的方案。
- 第一阶段优先采用 Streamlit 或轻量 Web UI；在技术选型确定前避免引入复杂前端工具链。
- 新增功能时同时补充最小可复现示例、必要测试和 README 使用说明。
- 路径处理必须使用 `pathlib`，不得硬编码其他电脑上的绝对路径；Hook 启动路径因 Codex 配置需要可例外。
- 尽量保持模块小而清晰；不要为了展示技术而增加无关框架。

## 验证要求

- 修改 Python 代码后，至少运行相关单元测试或最小 smoke test。
- 修改 UI 后，启动本地应用并实际检查主要页面和交互。
- 修改 Hook 后，分别测试允许的根目录内路径和应拒绝的根目录外路径。
- 交付时列出完成内容、验证证据、已知限制以及用户下一步可以执行的命令。

## Git 与 GitHub

- 可以在本地创建分支和提交，但不得覆盖或丢弃用户已有改动。
- 未经用户明确确认，不得把仓库发布为公开仓库，不得向远程仓库推送。
- 第一次连接 GitHub 前，先检查待提交文件中是否包含隐私、密钥、公司资料或大文件。
- 提交信息使用简洁英文，例如 `docs: add project overview`、`feat: add experiment config diff`。

## 文档优先

- 对 Codex、GitHub、框架 API、模型接口等容易变化的内容，优先查官方文档。
- 如果官方文档与本机环境不一致，记录冲突，并采用在本机验证可用的做法。
