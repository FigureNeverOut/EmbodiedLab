# EmbodiedLab MVP 设计

## 第一条纵向功能

第一版 MVP 比较两份 JSON/YAML 实验配置，输出人类可读的 Markdown 报告或便于后续程序处理的 JSON 报告。

选择它作为第一步的原因：

1. 它来自真实的具身智能复现痛点，不是为了凑功能。
2. 不需要 GPU，也不依赖公司数据。
3. 输入、处理、输出边界清晰，适合建立测试和项目结构。
4. 后续可以自然扩展到实验数据库、Web UI、日志曲线和失败 episode。

## 用户故事

作为一名复现实验的研究者，我希望选择一份 baseline 配置和一份 candidate 配置，快速看到新增、删除和改变的字段，避免因为任务数、batch size、loss weight 或模型 backend 不一致而错误归因实验结果。

## 当前输入与输出

输入：

- `.json`
- `.yaml`
- `.yml`
- 文件根节点必须是键值映射。

输出：

- Markdown 差异表，适合阅读或放入实验记录；
- JSON 结构化报告，适合后续接入 UI 或数据库；
- 输出路径强制位于当前 Git 项目根目录内。

## 差异类型

- `added`：字段只存在于 candidate；
- `removed`：字段只存在于 baseline；
- `changed`：字段值或类型发生改变；
- `unchanged`：可选显示，表示字段完全一致。

## 暂不包含

- 配置 schema 语义校验；
- 浮点容差；
- 数组逐元素比较；
- Streamlit 图形界面；
- 实验数据库。

这些能力将在核心比较逻辑稳定后按优先级加入。
